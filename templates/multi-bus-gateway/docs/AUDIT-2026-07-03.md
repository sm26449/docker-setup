# Full Audit & Evaluation — 2026-07-03

Complete evaluation of the UI and the codebase, from two lenses: a **first-time
user** opening the app, and an **authorized technician** (expert in Fronius,
Victron, Moxa, Teltonika). Goal: identify what is missing / would help, and where
to invest to go *far above* any commercial product.

Grounded in a four-way code survey (protocol engine, device/vendor integration,
frontend/UX, code/security/reliability). Extends the internal
[`docs/vendor/GAP-ANALYSIS-2026-07.md`](vendor/GAP-ANALYSIS-2026-07.md) (which
covers gateway *feature parity*) into **catalog breadth, UX, and
security-by-default**.

## Verdict

The *machinery* is genuinely strong — in places it already beats commercial
gateways (the reverse-engineered Fronius native-CG emulation, the store-and-forward
guarantee, the vmeter watchdog, the Status/observability layer). What holds it
back is **breadth** (device catalog), **depth** (write path + protocol coverage),
**UX polish** (onboarding, half-applied i18n, no bulk ops), and **security
defaults**. None of it is architecturally hard — the plumbing to fix most of it
already exists.

## 4. Domain expert — device catalog & vendor depth

Catalog is essentially empty for a "gateway": **3 input templates** (Janitza + 2
*draft* Fronius meters) + **3 output emulations**. Machinery exists; content doesn't.

- **Missing common meters** (day-one expectations): Eastron SDM120/220/630, Carlo
  Gavazzi ET112/EM540 (Victron's *current* recommended meters — you emulate the
  aging EM24 but not these), Schneider iEM/PM, ABB B-series, Socomec. (Competitor
  JUG25 already ships ABB B21/B23/B24.)
- **Fronius = grid meter only, legacy platform only** — no inverter data, no
  battery/storage, no `GetPowerFlowRealtimeData`, **no GEN24 SunSpec** (unit
  hardcoded to DataManager's 240), no Ohmpilot, no Wattpilot.
- **Victron = one emulated model, zero reads** — EM24-AV53 out only; no Cerbo/
  VE.Bus reads (SoC/ESS/DVCC), no ET340/EM540/SDM emulation.
- **Teltonika = analyzed as a competitor, zero device integration** — no RMS,
  cellular/signal, GPS, digital I/O, router health (all exposed via Modbus-TCP/
  JSON-RPC — a fit for the existing drivers).
- **Whole ecosystems absent**: SolarEdge/Huawei/SMA inverters (SunSpec-TCP),
  Pylontech/BYD/Seplos batteries, EVSE. No SunSpec input model library, no
  template registry/sharing.

## 5. Code, security, reliability, ops

**Strengths (near production-grade):** corrupt-config guard (`.yaml.bad` +
save-block); store-and-forward with idempotent replay; vmeter freshness watchdog
+ wedged-thread force-restart; bounded resources everywhere; well-reasoned health
semantics. Reliability is not the weak spot.

**Real risks (prioritized):**
1. **Insecure by default** — API (incl. control endpoints feeding an ESS) is
   unauthenticated over plain HTTP; auth/API-key/allowlist/TLS all opt-in. `api.py:272`
2. **Plaintext secrets** — `mqtt.password` + `influxdb.token` cleartext in
   config.yaml (UI passwords *are* PBKDF2-hashed → inconsistency shows it's fixable). `config.py:775,795`
3. **Weak default admin creds** (`admin/admin`), no forced change. `config.py:138`
4. **No audit log** — device/sink transitions logged, but not logins/config
   changes/backup imports (compliance gap).
5. **Non-atomic config write** — `config.yaml` saved with plain `open('w')`
   (register files use atomic `os.replace`); crash mid-write can truncate live
   config. No version history/rollback. `config.py:848`

**Nice-to-haves:** no Prometheus `/metrics`, no OTA, dependency floor-pinning
(only pymodbus exact-pinned; no lockfile/SBOM); `alerts.py` + `event_log.py` +
`backfill.py` untested (the alert path is what you'd rely on during an outage).

## 6. Roadmap — to go far above commercial

**Tier A — close the credibility gaps (highest leverage)**
- Write path (FC6/16 setpoints/resets) — bidirectional gateway.
- RTU slave + TCP↔RTU bridge — the "gateway" promise.
- Device-catalog library — bundle 15–20 real maps + CSV & SunSpec-model import + Modbus/SunSpec auto-scan.
- Security-by-default — forced first-boot credential, secret encryption-at-rest, HTTPS on.
- Fix P4 (byte-order matrix + little-endian int) and the atomic config write — small, real wins.

**Tier B — UX that beats commercial**
- Real onboarding wizard, finish i18n, surface Monitor/History/Energy, bulk register ops, CSV/PNG export, Prometheus `/metrics` + auto-provisioned Grafana.

**Tier C — ecosystem depth**
- Fronius inverter/battery/PowerFlow + GEN24 SunSpec; Victron Cerbo reads; Teltonika telemetry template; EVSE/battery library; template registry/sharing.

**Tier D — ops maturity**
- Config versioning + rollback, audit log, OTA, SBOM/lockfile, tests for the alert/event paths.
