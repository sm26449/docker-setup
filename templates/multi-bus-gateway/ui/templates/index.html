    Bus Monitor

    Frame-level Modbus trace — the exact bytes on the wire, one row per transaction (every retry included). Runtime only: the buffer lives in memory and capture stops on restart.

    Time Device Unit FC Addr ×count Result ms Proto

    Register Probe

    One-shot read on any Modbus device with every interpretation side by side — data type × word order (ABCD / CDAB / BADC / DCBA). Read-only. With the capture running, the probe's frames show up in the monitor above, exception replies included.

    SunSpec Scan

    Walks the SunSpec model chain (the SunS marker at 40000/50000/0, then every model the device declares) — identity, meters, MPPT, storage. Read-only; works on most Fronius / SMA / SolarEdge / Huawei / ABB inverters and SunSpec meters.

    General

    IANA name (e.g. Europe/Bucharest). Sets the month boundaries for the Energy report — applies immediately, no restart.

    Default widget colors

    Applied to NEW widgets only — existing choices are kept. "Reapply defaults" lives in the dashboard's Customize dialog.

    Backup & Restore

    Download a ZIP with all devices, measurement selections, device templates and virtual meters — or restore one. Secrets and network settings are excluded by default.

    Snapshots & Rollback

    A snapshot of the full config is taken automatically after every change (devices, measurements, templates, virtual meters, settings). Restore rolls everything back to that moment — the current state is snapshotted first, so a rollback is itself reversible. “Last known good” updates only after a healthy boot and restores automatically if the config no longer loads.

    Login

    The operator can use the commissioning tools (trace, probe, discovery) and write to devices within template bounds, but cannot change any configuration.

    Passkeys

    Password-less sign-in bound to this device (fingerprint, PIN, security key). Browsers require a secure context: register and sign in via localhost or an HTTPS hostname — a plain-HTTP IP address will not work. You can enroll now, before enabling login.

    Name Account RP Created

    Passwords are stored hashed. Leave a password blank to keep the current one. Applies immediately.

    Modbus writes

    Off by default — writing to real hardware is irreversible. When on, only registers declared writable in a device template can be written, within their bounds, and every write is confirmed, authenticated and audited. The primary device stays read-only.

    Audit Trail

    Every configuration change, login, device write and secret export — who, from which IP, when, with the payload redacted. With auth on, only the admin can read this.

    Alerts

    Infrastructure-health alerts (a device or sink drops, latency spikes, the InfluxDB buffer backs up) delivered over MQTT and/or an HTTP webhook. Not value alerts. Applied live on save.

    Signals

    Channels

    Empty = webhook off. POSTs the alert JSON here (no LAN restriction — meant for external notifiers).

    Placeholders: {severity} {source} {message} {key} {host} {ts}. Empty = raw alert JSON.

    Virtual Meters

    Serve a device's live values as a standard Modbus meter so other systems read them natively (EM24 → Victron grid meter, Fronius Smart Meter → DataManager). Enable only to validate in parallel with the real meter — this feeds a control-critical reference.

    Templates

    The catalog behind your devices. Device maps tell the gateway how to read a real meter (register addresses, function code, byte order). Meter emulations re-serve live values as a standard Modbus meter. Bundle, upload, import from CSV, edit and share them here.
